# Bounty program snippets

Copy-paste building blocks for operating the Simpler Grants Funded Open Source
Contributor Program. These are the canonical text fragments referenced by the
program SOP; keep them in sync with the SOP and the
[Contributor Terms of Service](/documentation/bounties/paid-contributor-TOS.md).

| Snippet | SOP reference | Use |
| --- | --- | --- |
| [`contributor-tracking-table.md`](./contributor-tracking-table.md) | Appendix A | The table that drops into every bounty issue description. |
| [`paid-banner.md`](./paid-banner.md) | Part 2.6 | Banner that replaces the bounty banner once an issue is paid and closed. |
| [`tos-acknowledgment.md`](./tos-acknowledgment.md) | Part 4.9 | Exact text a contributor must post before `/claim`. |
| [`operator-replies.md`](./operator-replies.md) | Parts 2, 5, 6 | Program-owner reply templates for the full claim → payout lifecycle. |

**Conventions used in these snippets**

- `@handle` — the contributor's GitHub handle.
- `#NNN` — the bounty issue number.
- `$AMOUNT` / `TIER` — bounty amount and tier (XL / L / M).
- `<date>` — ISO `YYYY-MM-DD`.
- Replace every placeholder before posting. Keep the program's voice: respectful,
  specific, and grounded in a cited rule.
