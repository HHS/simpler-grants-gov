# Bounties

The Simpler.Grants.gov project offers bounties to encourage and reward meaningful contributions from the open-source community. Bounties are tied to specific GitHub issues and provide a way for external contributors to get compensated for their work.

## How bounties work

1. **Find a bounty issue** — Look for issues labeled with a bounty tier (e.g., `bounty:L`, `bounty:M`, `bounty:XL`) in the [GitHub issue tracker](https://github.com/HHS/simpler-grants-gov/issues).
2. **Review the acceptance criteria** — Each bounty issue includes clear acceptance criteria, scope, and suggested starting points.
3. **Claim and work on the issue** — Fork the repository, create a branch, and submit a pull request that addresses the acceptance criteria.
4. **Get your PR reviewed and merged** — Work with the maintainers to get your changes approved and merged.

## Bounty tiers

| Tier | Complexity | Description                                                                 |
| ---- | ---------- | --------------------------------------------------------------------------- |
| L    | Low        | Small, well-defined tasks such as documentation updates or minor bug fixes. |
| M    | Medium     | Moderate tasks that may involve changes across a few files or components.   |
| XL   | Extra Large| Larger tasks that require significant development effort or design work.    |

## Acceptance criteria

Every bounty issue includes an **Acceptance criteria** section. To have your contribution accepted, you must:

- Meet all of the acceptance criteria listed in the issue.
- Ensure all existing tests continue to pass and add new tests that cover the changed behavior.
- Update documentation where the change is user-visible or developer-facing.
